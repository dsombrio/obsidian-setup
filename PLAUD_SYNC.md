# Plaud Sync — Bridging Plaud to Obsidian

Plaud captures voice notes and creates tasks automatically. The plaud-sync script reads those tasks from the SQLite database and writes them into Obsidian in a format Dataview and the Tasks plugin can read.

---

## How It Works

```
Plaud App (iPhone/Mac)
    ↓ records voice, transcribes, creates tasks
SQLite Database (/Users/tradbot/Library/Application Support/TradBot/crm.db)
    ↓ read by
plaud-sync.py (runs daily via cron)
    ↓ writes to
Obsidian Vault/04 Tasks/Plaud Tasks - YYYY-MM-DD.md
    ↓ embedded in
Daily Note (via ![[Plaud Tasks - YYYY-MM-DD]])
    ↓ readable by
Dataview + Tasks plugin
```

## Plaud Database Schema

**Database path:** `/Users/tradbot/Library/Application Support/TradBot/crm.db`

**Relevant table — `tasks`:**

| Column | Type | Description |
|---|---|---|
| `id` | INTEGER | Primary key |
| `user_id` | INTEGER | User (always `1` for David's instance) |
| `contact_id` | INTEGER | Linked contact (nullable) |
| `deal_id` | INTEGER | Linked deal (nullable) |
| `title` | TEXT | Task description |
| `description` | TEXT | Full transcription or notes |
| `due_date` | DATE | Due date if set (nullable) |
| `timeframe_minutes` | INTEGER | Estimated time (e.g., `480` = 8 hrs) |
| `source` | TEXT | `plaud` (AI-generated) or `manual` |
| `status` | TEXT | `pending` or `completed` |
| `created_at` | DATETIME | When task was created |
| `completed_at` | DATETIME | When marked done (nullable) |
| `priority` | TEXT | `high`, `medium`, or `low` |

**Status:** Currently 194 tasks total. 48 Plaud-generated + 1 manual = 49 shown as examples in the sync.

---

## plaud-sync.py — What It Does

1. Connects to the SQLite database
2. Reads all tasks where `source = 'plaud'` and `status = 'pending'`
3. Formats them as Obsidian Tasks-plugin-compatible checkboxes with frontmatter
4. Writes to `04 Tasks/Plaud Tasks - YYYY-MM-DD.md`
5. Also writes a `Plaud Tasks Archive - YYYY-MM-DD.md` (optional full history)
6. Logs sync results with timestamps

## Configuration

Edit the top of `plaud-sync.py` to match your setup:

```python
# === CONFIGURATION ===
PLAUD_DB = "/Users/tradbot/Library/Application Support/TradBot/crm.db"
OBSIDIAN_VAULT = "/Users/tradbot/Library/Mobile Documents/iCloud~md~obsidian/Documents/Tradition Sales"
TASKS_FOLDER = "04 Tasks"
DAILY_FOLDER = "00 Daily"
# === END CONFIGURATION ===
```

**How to find your vault path on iPhone:**
1. Open Obsidian on iPhone
2. Settings → About → Vault location
3. Use the iCloud path shown there

**How to find your vault path on Mac:**
1. Open Obsidian → Settings → Vault
2. The path is shown as the vault folder location

## Cron Setup (Mac)

Run the sync every morning at 6:00 AM before you start:

```bash
# Add to crontab
crontab -e
```

```
0 6 * * * /usr/bin/python3 /Users/tradbot/.openclaw/workspace/scripts/plaud-sync.py >> /Users/tradbot/.openclaw/workspace/logs/plaud-sync.log 2>&1
```

Or run via launchd for background reliability (see `SCRIPTS/com.traditionsales.plaud-sync.plist`):

```bash
# Install the launchd agent
cp SCRIPTS/com.traditionsales.plaud-sync.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.traditionsales.plaud-sync.plist
```

**Make the log directory first:**
```bash
mkdir -p /Users/tradbot/.openclaw/workspace/logs
```

## iPhone Sync

On iPhone, the script won't run natively. Options:

1. **Run on Mac Mini** (always on) via cron or launchd — this is the recommended setup
2. **Run via iOS Shortcuts + Mac (via SSH):** Use a Shortcuts automation to SSH to the Mac Mini and run the script (requires SSH key auth setup)
3. **Run manually:** Open the Mac Mini terminal and run the script when needed

**Recommendation:** The Mac Mini should run this as a cron job every morning. The iPhone vault syncs via iCloud automatically. By the time David opens Obsidian on his iPhone each morning, the Plaud Tasks note is already there.

## Querying Plaud Tasks with Dataview

Once tasks are written to Obsidian, query them anywhere:

### All pending Plaud tasks
```dataview
TASK
FROM "04 Tasks"
WHERE source = "plaud" AND status = "pending"
```

### High priority Plaud tasks
```dataview
TASK
FROM "04 Tasks"
WHERE source = "plaud" AND status = "pending" AND priority = "high"
```

### Tasks due this week
```dataview
TASK
FROM "04 Tasks"
WHERE due <= date(today) + dur(7 days) AND status = "pending"
SORT due ASC
```

### Tasks by source (Plaud vs manual)
```dataview
TABLE source, length(rows) AS Count
FROM "04 Tasks"
WHERE status = "pending"
GROUP BY source
```

## Manual vs Plaud Tasks

All tasks have a `source` field:
- `source: plaud` — created by Plud's AI
- `source: manual` — created directly in Obsidian

This distinction matters for attribution. When a task from a Plaud recording about a sales call gets completed, the related deal note should be updated.

---

## Error Handling

The script handles these cases gracefully:
- Database doesn't exist → log error, exit cleanly
- No pending tasks → write empty file with "No pending Plaud tasks" message
- Database locked → retry once after 2 seconds
- Vault folder doesn't exist → create it
